# 05 · P3a 公告派息数据源（除息前即更新）

> 目标：在公司**宣派**新分红、**除息日之前**就把真实的金额与日期抓进来，
> 让「即将到账 / 预计全年」用**真实已公告数据**替代「按去年节奏的预估」。
> 这是 04-P3「方向六：公司公告数据源」的细化实施方案。

## 0. 背景与现状

当前股息数据有两层：

- **真实已发生**：`update_market_data.py` 从 yfinance 拉每只股票近 5 年的**已除息**派息（除息日 + 每股金额），写进 `market.json` 的 `quotes[symbol].dividends[]`。除息后约 1 天内被抓到。
- **未发生的**：前端 `buildForecastDividendEntries` 用**去年同期金额**做「节奏预估」（灰点）占位。

**缺口**：公司宣派到除息之间这段时间（少则数日、多则数周），我们拿不到真实新政策，"即将到账/预计全年"显示的是**去年的旧金额**。若今年分红翻倍/腰斩/特别派息，会失真，直到除息日过后才自动修正。

**本方案要补的就是这段**：抓「**已公告、未除息**」的派息（declared dividends）。

## 1. 数据源选型

没有免费官方结构化 API，按市场分别取，**全部按"尽力而为的增强"对待，失败不影响既有数据**：

| 市场 | 推荐主源 | 关键字段覆盖 | 备注 |
|---|---|---|---|
| A 股 | **东方财富数据中心·分红送配**（datacenter-web `RPT_SHAREBONUS_DET` 一类接口） | 公告日 / 股权登记日 / **除权除息日** / 派息日 / 每股股利 / 是否实施 | 结构化 JSON、覆盖最好；字段名需 CODEX 实测确认 |
| 港股 | 东方财富·港股分红 或 etnet / AAStocks 分红页 | 除净日 / 派息日 / 每股股息 / 货币 | 覆盖与稳定性次于 A 股，可能要解析页面 |
| 美股 | yfinance `ticker.calendar` / `get_info()['dividendDate','exDividendDate']` | 下一次除息日 / 派息日 / 金额 | 美股相对可靠 |

> ⚠️ 诚实提示：以上多为非官方接口，无 SLA、可能改版失效，需要监控与容错。港股「已宣派未除息」的覆盖天然不如 A 股。**确切的接口地址和字段，交给 CODEX 实测核对，不要照搬本文档的猜测命名。**

### 日期语义务必对齐（最易错）
- `exDate`（我们口径的除息日）= **除权除息日**，不是股权登记日。
- `payDate` = **派息日 / 红利发放日**。
- A 股「预案」与「实施」要区分：预案金额可能被股东大会改动，需标注 `tentative`。

## 2. 数据模型变更

### 2.1 公开 `market.json` 的派息事件扩展
给 `quotes[symbol].dividends[]` 每个事件加字段：

```jsonc
{
  "exDate": "2026-08-20",
  "payDate": "2026-09-10",
  "amountPerShare": 0.62,
  "currency": "HKD",
  "source": "eastmoney",        // 数据来源
  "status": "announced",        // 新增：paid | ex | announced
  "announceDate": "2026-06-28", // 新增：公告日（可空）
  "tentative": false            // 新增：A股预案未实施=true（可空）
}
```

- `status: "announced"` = 已公告、`exDate` 在未来。来自公告源。
- `status` 缺省（旧数据）按 yfinance 的"已除息历史"处理，等价 `ex`/`paid`。
- 同一只股票，**已公告**事件与未来 yfinance 抓到的**已除息**事件是同一笔，需要去重（见 §4）。

### 2.2 私有 `portfolio.json`
**不需要新增结构**。已公告但未除息的派息 `exDate > today`，结算（`revenue.js`）本就不会建账本条目，只供前端「即将到账」展示。除息日一过，走现有真实入账流程。

## 3. 脚本改动（`scripts/`）

1. 新增 `fetch_announced_dividends.py`（或在 `update_market_data.py` 内新函数）：
   - 入参：watchlist / portfolio 的 symbol 列表。
   - 按市场路由到对应源，返回「已公告未除息」事件列表。
   - 字段标准化：统一成 §2.1 的结构，`status="announced"`，日期映射到 exDate/payDate。
2. 合并进 `market.json`：
   - 与现有 yfinance 历史事件**按 (symbol, 归一化金额, exDate 邻近) 合并去重**。
   - **容错**：公告源失败 → 保留既有 `dividends[]` 不动（与现有 dividend 容错一致）。
3. 频率：随 `update-market-data` 一起跑（现为每天 3 次）。

## 4. 前端改动

### 4.1 即将到账逻辑（`compute.js`）
`computeDividendCalendar` 的「即将到账」来源升级为三类，优先级从高到低：

1. **在途待到账（pending）**：已除息、有效到账日在未来（现有）。
2. **已公告未除息（announced）**：来自 `market.json` 中 `status="announced"` 且 `exDate > today` 的事件 × 当前持股。**真实金额/日期**。
3. **节奏预估（forecast）**：仅当某 (symbol, 到账月) **既无 announced 也无账本** 时才用（现有逻辑作兜底）。

去重：若某 (symbol, 到账月) 已有 announced，则跳过该月的 forecast（与现有 `ledgerMonthKeys` 跳过 forecast 同理，扩展为 `ledger ∪ announced`）。

### 4.2 展示（`render.js` / 月份弹窗）
新增第四种状态色，区分"可靠的已公告"与"纯节奏猜测"：

- 🟢 绿 `is-confirmed`：已确认到账（现有）
- 🟡 黄 `is-unconfirmed`：自动入账待确认（现有）
- 🔵 蓝 `is-announced`：**已公告未除息（真实金额）**（新增）
- ⚪ 灰 `is-forecast`：节奏预估（现有，兜底）

`announced` 行同 `forecast` 不可点击确认（还没到账）。可在日期处显示「除息 08-20 · 派息 09-10」真实日期，不带"(估)"。

## 5. 阶段拆分

- **P3a-1 · A 股公告派息**：东方财富分红送配接口，最高性价比、数据最干净。先打通脚本 → market.json → 前端 announced 展示。
- **P3a-2 · 港股公告派息**：东方财富港股分红 / etnet 解析，覆盖核心持仓即可。
- **P3a-3 · 美股**：yfinance calendar/dividendDate。
- **P3a-4 · 前端 announced 态打磨**：蓝点、真实日期、与 forecast/realized 的三方去重与过渡。

## 6. 验收标准

- `market.json` 的 `dividends[]` 含 `status="announced"` 的未来事件，日期/金额与公司公告一致（抽样核对几只）。
- 某持仓有已公告未除息派息时，「即将到账/预计全年」用**真实金额**，月份弹窗显示**蓝点 + 真实除息/派息日**，不再是去年金额的灰点预估。
- 公告源拉取失败时，`market.json` 既有派息数据不被清空。
- 除息日过后：announced 事件被 yfinance 的真实历史接管，自动转入账本，**不重复计数**。
- 已确认（绿点）条目不受影响。
- `npm run build` 通过。

## 7. 风险与边界

- 非官方源无 SLA，可能改版；需日志监控与失败告警。
- 港股覆盖不全；覆盖不到的仍回落到节奏预估（不退化于现状）。
- A 股「预案 vs 实施」「送转 vs 现金」要正确过滤——本方案只取**现金分红**。
- 不在本期范围：送股/转增、配股、要约等其它公司行为。
