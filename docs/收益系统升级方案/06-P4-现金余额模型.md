# 06 · P4 现金余额模型

## 目标

让「出入金」和「交易」两个按钮拥有真实意义:引入一个**现金余额**,使净值 = 股票市值 + 现金 − 负债。这样资金收益自动包含股息,不需要为每一笔股息手动调整,也不会重复计算。

## 模型（现金余额驱动）

```
净值 = 股票市值 + 现金余额 − 负债
现金余额 = 期初现金
         + 入金 − 出金            （cashFlows，算净流入）
         + 卖出所得 − 买入花费      （trades，不算净流入）
         + 已到账股息              （dividendLedger 中已确认/已改实收金额的，不算净流入）
资金收益 = 当前净值 − 去年末净值 − 净流入
收益率   = 资金收益 ÷ 去年末净值
```

净流入只含**真实的银证转入/转出**(出入金)。股息、买卖都是账户内部,不进净流入,由现金余额和持仓自动吸收。

## 事件对照

| 事件 | 现金余额 | 持仓股数 | 净流入 |
|---|---|---|---|
| 收到股息(已到账) | + | 不变 | 否 |
| 入金 | + | 不变 | 是 |
| 出金(股息不复投花掉) | − | 不变 | 是 |
| 买入 | − | + | 否 |
| 卖出 | + | − | 否 |

## 关键规则

- **持仓股数只读**:主页不再直接改数量,只能通过「交易」买入/卖出调整。有效股数 = 期初股数 + 期初日之后的买卖净额。
- **股息自动进现金**:股息在日历里被标记已到账(或填了实收金额)后,自动计入现金余额,无需手工搬。
- **期初 = 现在**(本次选择):保留当前持仓股数作为期初持仓,用户填一次「期初现金」即启用;设定日(`openingDate`)之前的交易/股息/出入金不重复计入。历史年份仍用 `yearlyManual` 的年末净值。

## 数据字段

私有快照新增:

```jsonc
openingCashCny: 0,     // 期初现金
openingDate: ""        // 期初日期；为空 = 未启用现金模式（行为与旧版一致）
```

## opt-in（零回归）

`openingDate` 为空时,现金模式**未启用**:现金=0、净值=股票市值−负债、数量可直接编辑,完全等于旧行为。用户填写期初现金后才切换到现金模型。

## 建议改动文件

- `src/state.js`（字段 + 持久化 + 同步快照）
- `src/compute.js`（`computeCashBalance`、有效股数、净值含现金）
- `src/modal.js`（期初现金弹窗；数量编辑改走交易）

- `index.html` / `styles.css` / `src/main.js`（入口与接线）

## 验收标准

- 未填期初现金时,一切与旧版一致。
- 填写期初现金后:净值 = 股票市值 + 现金 − 负债。
- 收股息(标记已到账)→ 现金增加,净值不变。
- 买入 → 现金减、股数增;卖出 → 现金增、股数减;净值只随价格波动。
- 入金/出金 → 计入净流入;股息与买卖不计入净流入。
- 主页数量只能通过交易调整。
- `npm run build` 通过。
