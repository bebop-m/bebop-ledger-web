# 00 · 总览与口径

## 核心判断

波普记账现在是 **当前持仓 + 年化预估**。要支撑「已收股息 / 即将到账 / 年度收益 / 历年对比」，必须新增 **历史事件账本**。

已收股息一律以事件为准，不再用 TTM 反推。首页现有的股息数字改名为 **预估年化股息**，与真实到账股息区分。

## 产品结构

底部导航固定为：

1. **资产**：现有首页，不继续复杂化核心仓 / 打工仓。
2. **收益汇总**：年度复盘。
3. **股息日历**：当年股息现金流。

核心仓 / 打工仓在新页面只作为筛选或小计出现：全部 / 核心仓 / 打工仓。

## 数据边界

公开仓库只放公共行情和公共派息数据：

- `data/market.json`
- `data/watchlist.json`
- `data/override.json`

私有仓库保存个人真实数据：

- 当前持仓
- 股息账本
- 每日快照
- 出入金
- 历史回填

## 统计口径

### 股息口径

v1 按 **实际到账日（有效 payDate）** 归属年份和月份统计。

由于数据源不提供 `payDate`，用「有效到账日」贯穿全链路：

```text
有效到账日 = 真实 payDate(若有/用户确认) → 按市场滞后估算(除息日 + lag) → 退回除息日
市场滞后(天)：A股 0，港股 30，美股 14   （见 constants.js: PAYDATE_LAG_DAYS）
```

- 估算出来的到账日在明细里标注「(估)」，并同时展示除息日。
- 港股「除息在 5 月、实际 6 月到账」的情况会正确落到 6 月。

页面文字需要明确：

```text
股息收入按实际到账日统计
```

### 即将到账口径

「即将到账」= **在途待到账(pending) + 节奏预估(forecast)**：

- `pending`：已除息、但有效到账日还在未来的账本条目（港股除息后的在途期）。
- `forecast`：按历史节奏推算的下一次，只展示、不写入 `dividendLedger`；
  且当同一 (symbol, 到账月) 已有真实账本条目时跳过，避免与实际派息重复计数。

「预计全年」= 已到账 + 即将到账（不再与首页「预估年化股息」并列，两者口径不同已分离）。

### 收益口径

收益汇总页 v1 不把「股息收入 + 资金收益」包装成唯一大数。

原因：

- 当前 App 不跟踪现金账户。
- 除息导致股价自然下跌，股息和净值变动会互相抵消。
- 复利再投资在没有交易流水前容易双算。

v1 展示：

- 股息收入
- 资金收益
- 合计参考值

合计参考值必须带口径提示。

## 关键字段约定

### 股息唯一键

不要只使用 `(symbol, exDate)`。同一天可能存在普通股息 + 特别股息。

建议使用：

```text
symbol + exDate + amountPerShare + currency
```

或生成稳定 `sourceId`。

### 到账与可信度拆分

不要把「是否到账」和「是否估算」混进一个 `status`。

建议使用：

```jsonc
receiptStatus: "received" | "pending",
confidence: "confirmed" | "snapshot" | "carryForward" | "estimated" | "manual"
```

### 冻结字段

股息入账时冻结：

- 除息日持股数
- 除息日汇率
- 当时税率
- 当时仓位类型

之后不随今日汇率、今日税率、今日仓位变化重算。
