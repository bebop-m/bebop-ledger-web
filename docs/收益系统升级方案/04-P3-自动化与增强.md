# 04 · P3 自动化与增强

## 目标

在 P1 / P2 已经可用的基础上，提高准确性、自动化程度和历史可回填能力。

## 方向一：到账确认 UI

为股息明细增加操作：

- 标记已到账
- 修改到账日
- 修改实收金额
- 添加备注

确认后：

```jsonc
confirmed: true,
receiptStatus: "received",
confidence: "confirmed"
```

自动结算不得覆盖确认条目。

## 方向二：历史回填 UI

增加历史年份回填入口：

- 年份
- 手动股息收入
- 年末净值
- 当年净注入

写入：

```jsonc
yearlyManual: []
```

用于补齐开始记录之前的年份。

## 方向三：出入金编辑 UI

增加出入金记录：

- 日期
- 金额
- 类型：入金 / 出金
- 备注

写入：

```jsonc
cashFlows: []
```

用于提升资金收益准确性。

## 方向四：私有仓库定时结算

可选增强：在私有仓库增加定时任务。

目标：

- 不打开 App 也能定期写快照。
- 定期结算股息 ledger。

注意：

- 私有 Action 需要读取私有 `portfolio.json`。
- 需要拉取公开 `market.json`。
- 仍然不能覆盖 `confirmed=true` 条目。

## 方向五：交易流水

若未来要准确计算：

- 已实现收益
- 持仓成本
- Yield on Cost
- 复利再投资归集
- 打工仓资金收益

需要新增交易流水：

```jsonc
trades: [{
  id,
  date,
  symbol,
  side,          // buy | sell
  shares,
  price,
  currency,
  fxRate,
  feeCny,
  bucket,
  note
}]
```

交易流水完成后，资金收益可以从「净值变化口径」升级为更精确的「交易 + 持仓 + 现金」口径。

## 方向六：公司公告数据源

当前 yfinance 对未来派息和到账日不稳定。

后续可研究：

- 港交所公告
- A 股交易所公告
- 公司 IR 页面
- 付费金融数据源

有稳定数据源后，`forecast` 可以升级为更可靠的「已公告未除息」。

## 建议改动文件

视具体方向拆分实施。

可能涉及：

- `src/modal.js`
- `src/state.js`
- `src/sync.js`
- `src/compute.js`
- `src/render.js`
- `styles.css`
- 私有仓库 workflow
- 新脚本

## 验收标准

按具体子方向单独验收。

共同要求：

- 不破坏 P1/P2 已有数据。
- 不覆盖用户确认过的 ledger。
- 旧快照仍可读取。
- 云同步仍可用。
- `npm run build` 通过。
