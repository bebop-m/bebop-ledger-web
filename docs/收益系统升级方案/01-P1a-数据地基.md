# 01 · P1a 数据地基

## 目标

建立收益系统的数据基础，但先不做完整新页面。

本阶段完成后，系统应该可以：

- 在 `market.json` 中保留逐笔派息数据。
- 在私有 `portfolio.json` 中保存股息账本、每日快照、出入金和历史回填。
- 每次打开 App 时自动写入当天快照。
- 根据逐笔派息和历史快照自动生成股息账本条目。
- 保持旧版快照兼容，不破坏现有持仓同步。

## 范围

### 公开 `market.json`

为每只股票增加逐笔派息数组：

```jsonc
quotes["00700.HK"].dividends = [
  {
    "exDate": "2026-05-14",
    "payDate": "",
    "amountPerShare": 0.5,
    "currency": "HKD",
    "source": "yfinance"
  }
]
```

要求：

- `update_market_data.py` 抓取窗口从当前逻辑扩展到 3-5 年。
- 保留现有 `dividendPerShareTtm`，不要影响首页预估年化股息。
- `payDate` 缺失时留空。
- 前端合并 quote 时保留并规范化 `dividends`。

## 自动结算算法

App 打开后执行：

1. 如果今天没有 `dailySnapshots`，基于当前持仓和当前汇率写一条快照。
2. 遍历当前持仓相关 quote 的 `dividends`。
3. 对每个 `exDate <= today` 且 ledger 中不存在同 `sourceId` 的派息：
   - 找到除息日当天或之前最近一次快照。
   - 找不到快照时用当前持仓估算，标记 `confidence="estimated"`。
   - 冻结 shares、fxRate、taxRate、bucket。
   - 计算 grossCny 和 netCny。
   - 写入 `dividendLedger`。
4. 不覆盖 `confirmed=true` 的条目。
5. 不覆盖用户手动修改过的条目。

## 快照保留策略

第一版可以先只追加，不做降采样。

如果实现降采样：

- 近 90 天保留每日。
- 更早保留每月末。
- 年末快照必须保留。

## 不做的事

本阶段不做：

- 底部导航。
- 股息日历页面。
- 收益汇总页面。
- 到账确认 UI。
- 出入金编辑 UI。
- 交易流水。

## 验收标准

- 旧版 `portfolio.json` / localStorage 能正常迁移。
- `buildPortfolioSnapshot()` 输出 version 2 且包含新增数组。
- 云同步不会丢失新增数组。
- `market.json` quote 保留 `dividends`。
- 打开 App 后会生成当天快照。
- 有历史派息时能生成 ledger 条目。
- 同一派息不会重复生成。
- `confirmed=true` 条目不会被自动覆盖。
- `npm run build` 通过。
