## 2. 全局排版纪律

- 字阶：首页 hero 47px/600/-1px；内页 hero 38px；节标 10px/600/字距 4–5px 全用 `--label`；正文行 12px/600；提示 10–10.5px。数字全部 `font-variant-numeric: tabular-nums`。
- 页头（内页）：grid `56px 1fr 56px`，左裸返回箭头（18px，`--muted`），中页名（11px 字距 5px），右按需（如「诊断 N」「分享」）。
- 选中态记号：4px 金点下划（`::after`）——月份、筛选、年份切换、币种、方向段选、速览当前项统一用它。
- 底部抽屉：顶圆角 22px、把手 32×3px `--track`、遮罩 `--mask`；动作行居中三键：删除(`--up`)/取消(`--hint`)/保存(`--ink`+金点)。
- 进度线语言：2px 高、`--track` 底、`--gold` 填充、线端 7px 金点；分段构成线同高（金/墨/褐/纸表类别）。
- 严禁：卡片边框、图标圆底托、chevron 装饰箭头（导航行不带）、发丝分隔线（分隔靠间距）。
- 隐私掩码、下拉刷新、左缘右滑返回、页面历史栈、云同步、toast 全部保留，仅换色。

## 3. 各页规格与功能对照（细节以画布为准）

### 首页（zen-final）
结构：brand+裸图标 → 净资产 hero（label 可点编辑负债）→ 本年股息（金线+6 月点，点月直达该月明细）→ 两行待办 → 五行纯文字导航（保留现有 HUD 摘要计算）→「＋记一笔」文字按钮。

### 持仓页
- hero=股票市值+当日涨跌；仓位=墨/金两段线+两个可点仓位（金点选中）+明细行（市值/税后年化/股息率）+组合行；
- 右上「诊断 N」（N>0 时 N 用 `--up`），页名下小字行同开诊断弹窗；排序=「按市值 ↓」文字按钮轮换（市值→股息率→年股息，同字段再点切向）；
- 逐股行两层，保留 4 个可点域（名称→详情/税后年化→税率/股息率→每股股息/市值权重→数量或现金模式交易）；**行内现价保留**；有在途/已公告股息的行名称后缀 4px 金点（tooltip 保留）；折叠展开保留。
- 子抽屉：持仓详情（台账七行+口径注）/ 诊断（严重红点·关注金点·数据质量灰点，只列异常）/ 单字段编辑（基准股数/税率/每股股息/负债共用：字距标题+口径注+金线大输入）。**税率纯手填，不加快捷项。**

### 股息日历页
- 筛选三仓金点；hero=预计全年+同比一行；金(已到账)/墨(在途)/纸(预估)三段构成线+三行图例（三桶互斥恒等于全年，沿用现有 pipeline 计算）；
- 12 月点阵：**三态**——已过月整体 opacity .42、当月加重+金点 5px、未到正常；月下标金额；**点月开当月明细抽屉**（点阵是唯一月明细入口）；
- 下部列表：待确认置顶（`--up`）+近期倒序（已到账金/在途灰）；
- 月明细抽屉：月份+合计、小结行、**收款进度金线**（已到账/当月合计，右标 %）、逐笔五态（已到账金/待核对红/在途灰/已公告墨/预估浅）、可点行进编辑；
- 股息到账抽屉：派付日/到账日/**入账币种三选（人民币/港币/美元）**/实收金额（按所选币种输入，金线大数字）/折算行（`按 {日期} 汇率 {rate} · 入账 ¥X`）/备注/金点确认行/删除。**存储仍只存 CNY netCny，折算仅输入辅助；第一版用当前 state.rates 折算并明示日期，历史汇率为后续增强。**

### 收益明细页
- hero=本年资金收益+率（净值链口径文案保留）；口径弱化为两行 `--hint` 小字（年初→当前·净注入；注明现金去了资金与交易）；
- 历年趋势=双线 SVG：零轴发丝线、墨褐线（`--line-soft`）=资金收益率**每点标带符号值**（红正绿负，负年下探零轴）、浅金线=股息收益率；图下「自 {起年} 累计年化 ±x.x% · 最深一年 −x.x%」（累计年化=各年简单收益率复利折年）；
- 年度行两层：主数行+（股息·年末/净注入）小字行；动作只剩「回填」（进行年无）；点整行进年度回顾；脚注保留 2021–2025 手工基准口径。

### 年度回顾页（整合快照+年鉴；删除 yearHoldings / yearAnnals 两个弹窗及入口）
顺序：年份切换（金点）→ **hero=本年收益率（区间简单法，XIRR 全部移除——UI 与 compute 的 xirr 计算一并删除）** → 四指标（股息收入/资金收益/净注入/当年交易）→ 归因：四段线+四行 **pp 拆分**（各项贡献率=金额/年初净值，四项相加=本年收益率；金额右列对照；**无注脚**；epsSplitCoverage<50% 时才显示一行覆盖提醒）→ 年度持仓：SVG 圆环（六阶 donut 色，环心=年份+项数）+图例（公司/占比/较上年增减金字）+已清仓与冻结口径一行 → 股息现金流 12 细柱 → 交易复盘（买红/卖绿）。
- 右上「分享」→ **分享卡**：仅比例与收益率（本年收益率大字、股息收益率+累计年化、归因 pp 拆分行、持仓占比 mini 环+前三名），**全卡禁止出现 ¥ 与绝对金额**；canvas 导出图片保存；水印「波普账本 · 比例已脱敏 · 无金额」。

### 资金与交易页
- hero=本年净注入（副行：入金/出金/笔数）；**现金余额新家**：hero 下次级焦点（¥值+基准日+金色「点击校准」→ openingCash 弹窗；收益明细页的现金入口删除）；
- 四类计数一行；三段流水（买卖/出入金/股息入账）各默认 3 行+「展开全部 N 笔」，行点击进对应编辑；金额统一墨色带符号，仅买(红)/卖(绿)词着色；
- 记一笔抽屉三选（交易/出入金/当前现金，保留"流水 vs 余额快照"文案）；交易抽屉（代码自动识别行+成交额自动折算行）与出入金抽屉照稿。

### 公司基本面页（v3 结构）
顺序：页名+注（年报口径·更新时间·按经营回报排序）→ **九家速览横滑条**（经营回报降序，名+%，当前金点，点击切换公司）→ 公司名（点开 picker 抽屉）+币种口径副行 → 公式块（≈x.x%/年+金/褐/纸三段+构成行+方法注两行+**组合锚**：组合加权 x.x%/年·本公司列 N 家中第 M，数据取 getPortfolioReturnSummary）→ **估值节**：市盈率行+股息率行，各配「贵⇄便宜」2px 标尺（左贵右便宜；股息率金点位=分位值；PE 金点位=100−分位；偏便宜/偏低时结论词 `--gold`，反之 `--hint`）→ 每股分红线图 → EPS 线图 → 年度明细折叠行 → 财报日历（自家金色，点行切换并回顶）。
- **PE 数据**：当前 PE=现价÷最新 EPS（币种经汇率归一）；**PE 历史分位需要历史年末价格序列，数据不足则该行只显示当前 PE 值、不画标尺**；股息率分位用现有 getDividendYieldPercentile。
- 分红/EPS 线图：**y 轴 min–max 自适应量程**（非零基线，因每点标值不构成误导；与收益率趋势的零轴规则不同，勿混用）。
- picker 抽屉：持仓按市值在前（右列市值+当前金点）、观察/已清仓在后。

## 4. 功能变更总表（相对现网）

新增：股息入账币种折算；月明细收款进度线；年度持仓饼图；分享卡；九家速览排行；估值节（PE+分位标尺）；组合加权锚；累计年化行；月点阵三态。
删除：XIRR（UI+计算）；yearHoldings/yearAnnals 弹窗；归因注脚（改条件式）；年度行快照/年鉴动作；基本面「下一场财报」单行；一切卡片框/图标底托/chevron。
迁移：现金余额入口 收益明细→资金与交易；月明细入口统一为月点阵。
不变：全部计算链（除 XIRR 删除）、数据管线、同步、诊断规则、税率纯手填语义、"已到账=手动确认"语义。

## 5. 施工纪律

1. 分支 `zen-redesign`；首个 commit 提交 designs/ 设计稿；之后**一页一 commit**（中文 commit message 循现有仓库风格）。
2. 顺序：token 层+全局（styles.css :root/夜间/抽屉/页头）→ 首页 → 持仓 → 股息日历 → 收益+回顾 → 资金与交易 → 基本面。
3. 每步跑通现有测试（见 package.json scripts / tests/），JS 改动最小化：优先改 render 层模板与 CSS；compute 层只动：XIRR 删除、币种折算辅助、PE/排行/锚数据组装。
4. 不改 data/、scripts/、sync、network 管线；manifest theme-color 更新为素禅底色。
5. **禁止 push、禁止合并 main**——完工停在分支上等验收。
6. 自测用测试与 node 静态检查，不占用浏览器预览（预览验收由主会话完成）。

## 6. 验收标准（主会话执行）

- 七页+全部抽屉与画布逐一对照（结构/字阶/金点记号/三态/进度线）；日夜两态 token 切换正确；
- 功能回归：四个可点域、诊断计数、排序轮换、月点阵下钻、币种折算入账、回填、分享卡无金额、picker/速览切换、隐私掩码全局生效、下拉刷新与边缘返回不回归；
- 测试全绿；无 console 错误；真机（用户）最终确认后合并 main 并推送（推送即触发 Pages 部署）。
