## 全局纪律（每个界面都受约束）
- 结构靠排版不靠容器：**无卡片边框、无图标底托、无 chevron、无分隔 rule**；分区只用留白。
- 字阶：首页 hero 47px/600；内页 hero 38px/600 居中；节标 10px/600/字距 4-5px var(--label)；页名 11px/字距 5px 居中；数字 tabular-nums。
- 页头统一 grid(56px/1fr/56px)：左裸返回箭头 18px，中页名，右功能位。
- 金色=唯一记号色（股息+选中）；选中态=4px 金点下划；红涨绿跌保留。
- 抽屉：22px 顶圆角、32×3 把手、居中文字三键（删除=up/取消=hint/保存=ink+金点）。
- 进度/构成线：2px 高、track 底、实色段、端点 7px 金点。
- 交互契约不许丢：隐私掩码、下拉刷新、左缘右滑返回、页面历史栈、云同步、诊断规则、税率纯手填、「已到账=手动确认」。

## 施工流程
1. 在 `zen-redesign` 分支继续（或从它新开）；**一界面一 commit**（中文信息循仓库风格）；测试全绿（npm run test:js / test:py；删 XIRR 时同步改 tests/annals.test.mjs 断言到本年收益率口径）。
2. 不动 data/、scripts/、sync/network 管线。
3. **禁止 push / 禁止合并 main**。

## 验收流程（强制）
每界面：施工完成 → 截图与该文件夹 定稿图.html **并排逐元素目检**（日/夜两态）→ 按 验收标准.md 勾清单 → 向用户出示对比截图确认 → 才进入下一界面。DOM/computed 采样不能替代目检。全部通过后由主会话合并推送。
