## 定稿图怎么落地（施工前必读）

定稿图是**一张 390px 宽画布上的静态快照**，而应用跑在 **iPhone 15 Pro Max（430×932）的单屏 PWA** 里。三处系统性差异，照搬绝对值必错：

### 1. 尺寸要等比缩放，不能照搬
定稿图的 47px / 214px / 58px 都是 390 画布下的值。直接写死，在 430 屏上等于把整套设计缩小 9%（首页因此返工过一次）。

**做法**：每个尺寸折算成 vw，再用 clamp 兜住两端。
```css
/* 定稿值 N（基于 390 画布）→ */
font-size: clamp(N*0.92 px, N/390*100 vw, N*1.35 px);
/* 例：47px → clamp(43.24px, 12.0513vw, 63.45px) */
```
- 390 屏精确回到定稿值，430 屏等比放大 1.103，窄屏（SE）不小于 0.92 倍，桌面不超过 1.35 倍。
- **没有全局缩放变量可用**：CSS 的 `calc()` 里长度不能除以长度得到纯数，`calc(100vw / 390)` 得到的仍是长度，乘上 px 会变成非法的 px² 让整条规则失效。只能每个尺寸各自折算。
- 媒体查询里的断点（如 `max-height: 760px`）是判据不是尺寸，**不要缩放**。

### 2. 真机有状态栏，定稿图没有
`apple-mobile-web-app-status-bar-style` 是 `default`（不透明），页面**从状态栏下方开始**，此时 `env(safe-area-inset-top)` 返回 0。定稿图顶部那 30px 是画在无状态栏画布上的，真机上状态栏已占 59pt 提供了视觉呼吸，**再叠 30px 就是重复留白**（首页因此返工过一次）。

**做法**：顶部设计留白取定稿值的一半左右（首页用 14px），并保留 `max(设计值, env(safe-area-inset-*))` 以防日后改用 `black-translucent` 需要真正避让。
```css
padding-top: max(clamp(...), env(safe-area-inset-top, 0px));
```

### 3. 边距只由页面根元素一处控制
`body` 有 `padding: 14px 22px`、`.app-shell` 有 `max-width: 386px`，与页面自身的内距叠加后，430 屏上内容区只剩 294px（应为 378px）。**每个页面在自己的 `body[data-active-page="X"]` 作用域内让 body 与 app-shell 交出 padding 和 max-width**，边距由页面根元素独占。

### 4. 定稿图里有些效果是浏览器默认行为的产物
它们不是刻意设计，但**既然定稿图长那样，就是目标视觉**。典型：
- `.hero-label` 是 inline `<span>`，行盒被容器 16px 默认字号的 strut 撑到 23px（元素自身只有 14.5px）。落地若改成 block，行盒只剩 14px，整块竖向节奏就紧了 —— 需显式写 `line-height: 23px` 复刻。
- `<p>` 保留浏览器默认 margin。

**对不上时先量定稿图的 computed 值，别靠肉眼猜**。在定稿图页面里跑 `getComputedStyle` / `getBoundingClientRect` 拿到真实数字，再去对齐。

## 覆盖旧层的纪律（styles.css 有 1 万行历史层）

zen 层写在文件末尾靠后来居上取胜，但**只写"想改的属性"不够**——没写的属性会继续用旧层的值，而且往往在特定屏宽/屏高下才显形。首页实际踩到 7 处，全部是这个模式：

| 属性 | 旧层残留 | 后果 |
| --- | --- | --- |
| `display` | `.home-hero` 是 `flex/column/align-items:flex-start` | `text-align:center` 对 flex 子项排列无效，整区贴左 |
| `flex` | `.home-nav-item` 有 `flex:1` | 五行平分被撑开的容器，行高 49px（应 22px） |
| `height` | `.home-month` 写死 `42px` | 月点区偏高 |
| `min-height` | `.home-top` 有 `28px` | 顶行偏高，hero 整体下移 |
| `font-size` | `.home-month.is-current` 另设 `14px` | 当月字号变大撑高整行 |
| `border` | `.home-metrics` 有深色描边 | 违反「无分隔线」红线 |
| `::before` | `.home-month::before` 是旧月份轴刻度线 | 月号上方多出一排小竖线 |

**做法**：新写一个区块时，对每个元素显式声明 `display / flex / height / min-height / padding / margin / border / font-size`，并把 `::before`/`::after` 显式 `content: none; display: none`。宁可多写几行，也别让旧层从背后渗出来。

**排查手段**：改完在页面里遍历一遍，找出所有仍在绘制的边框与伪元素装饰：
```js
[...document.querySelectorAll('#page *')].filter(e => {
  const g = getComputedStyle(e);
  return parseFloat(g.borderTopWidth) > 0 && g.borderTopStyle !== 'none';
});
```

## 全局纪律（每个界面都受约束）
- 结构靠排版不靠容器：**无卡片边框、无图标底托、无 chevron、无分隔 rule**；分区只用留白。
- 字阶（均为 390 画布下的定稿值，落地按上面的等比规则缩放）：首页 hero 47px/600；内页 hero 38px/600 居中；节标 10px/600/字距 4-5px var(--label)；页名 11px/字距 5px 居中；数字 tabular-nums。
- 页头统一 grid(56px/1fr/56px)：左裸返回箭头 18px，中页名，右功能位。
- 金色=唯一记号色（股息+选中）；选中态=4px 金点下划；红涨绿跌保留。
- **红涨绿跌要覆盖到百分比**：金额与其后的涨跌幅同属一段、同色（首页 hero 用 `▼ -¥1,125 · -0.43%` 的 `·` 分隔式，不用括号）；导航摘要里的收益额与百分比同样着色，其余摘要保持 muted 叙述色。
- 抽屉：22px 顶圆角、32×3 把手、居中文字三键（删除=up/取消=hint/保存=ink+金点）。
- 进度/构成线：2px 高、track 底、实色段、端点 7px 金点。
- 交互契约不许丢：隐私掩码、下拉刷新、左缘右滑返回、页面历史栈、云同步、诊断规则、税率纯手填、「已到账=手动确认」。
- 展示的比例/百分比一律接实时计算链，**不许写死定稿图上的示意值**（定稿图的 58%、+5.7% 都是假数据）。

## 施工环境（新会话先看这段，都是踩过的）

- **dev server 用 `dev-alt`(5180)**，不要用默认的 5173——它常被并发会话占着。
  截图脚本默认就打 `http://localhost:5180`，端口对不上会截到空白页。
  launch.json 两个配置都加了 `--strictPort`，端口被占会直接报错而不是静默漂移
  （漂移正是此前预览面板截不到图的原因之一）。

- **源码全是 CRLF**（styles.css / src/*.js / index.html）。
  用 Edit 工具做**跨行**匹配几乎必然失败（old_string 里的 `\n` 对不上文件里的 `\r\n`），
  报错却看不出原因。跨行改动一律写个 node 脚本按行处理：
  ```js
  const s = readFileSync(F, 'utf8');
  const EOL = s.includes('\r\n') ? '\r\n' : '\n';
  const L = s.split(EOL);            // 按行定位、splice 替换，再 join(EOL) 写回
  ```
  单行替换用 Edit 没问题。

- **zen 层写在 styles.css 末尾**，靠后来居上压过前面一万行历史层。
  首页那段（`01-首页 · 按 designs/…/定稿图.html 重排`）是已验收的样板，
  新界面照它的写法：每个元素显式声明 display/flex/height/min-height/padding/
  margin/border/font-size，伪元素显式 `content: none; display: none`。

- **抽屉遮罩要去掉模糊**：现有 `.modal-mask` 一类带 `backdrop-filter: blur(...)`，
  而定稿图要求「背景页保持原位**被压暗**」——只有 `rgba` 遮罩，没有毛玻璃。
  03/04/05 及所有抽屉界面施工时一并清掉（styles.css 里搜 `backdrop-filter`）。

- **改完先跑 `npm run test:js`**（38 项）。渲染层改动一般不影响，但删算法/改口径时会。

## 施工流程
1. 在 `zen-redesign` 分支施工；**按界面簇推进**（见下方「按界面簇施工」），簇内一界面一 commit（中文信息循仓库风格）；测试全绿（`npm run test:js` / `test:py`；删 XIRR 时同步改 tests/annals.test.mjs 断言到本年收益率口径）。
2. 不动 data/、scripts/（截图脚本除外）、sync/network 管线。
3. **整簇**验收通过后合并 main 推送上线（Pages 只在 push 到 main 时部署）；合并前先 `git fetch` 把行情机器人的数据提交拉进来，避免覆盖。不要只上线宿主页面而把抽屉留在旧版。

## 验收流程（强制）

**顺序不能反：先出图用眼睛看，再量数值。** 数值只能验证你想到要验的维度，验不出你没想到的——首页曾出现「17 项样式属性全等、10 个纵向锚点 0px 偏差」却整个 hero 区左对齐的情况。

0. **先起 dev server**：`preview_start` 选配置 `dev-alt`（5180），或 `npm run dev -- --port 5180 --strictPort`。
   下面的命令都打 5180；没起的话脚本会立刻提示怎么起（不会卡超时）。
1. **机检 + 出图（一条命令）**：`npm run verify -- --page=<页>`
   自动跑完机械性红线（水平居中失效 / 描边 / 横贯分隔线 / 伪元素装饰 / 按钮底托 /
   紫色残留 / 数字等宽 / 单屏 / 横向溢出 / 尺寸等比 / 掩码破版），
   并出日夜两张图到 `screenshots/`。全绿才算过了红线，退出码非 0 即有未过项。
   页名：`home`、`holdings` 等 data-page-nav 值，抽屉写 `quickAdd@modal`。
2. **目检**：用 Read 看 PNG，与该文件夹 定稿图.html 并排逐元素对照，日/夜两态各一次。
   **机检管"有没有违规"，人眼管"像不像定稿图"——构图、字阶、留白节奏只能靠看。**
   需要更细的出图控制（叠状态栏、换视口、批量）用 `npm run shot`，见本目录 `截图与验收.md`。
3. **补测本界面清单**：验收标准.md 里那些与业务相关的项（数据口径、交互去向）机检覆盖不到，逐条手测。
4. 向用户出示截图确认，才进入下一界面。

> 机检本身也会漏。它最初只查了 `border-top`，漏掉持仓页页头那条只有 `border-bottom`
> 的 rule；居中检查最初把顶行图标、月点这类本就并排的元素误报成"未居中"。
> 所以：**机检全绿不等于验收通过，目检不可省**；反过来若机检报了项，先按报的修。

## 按「界面簇」施工，不要按编号逐个做

18 个界面不是 18 个独立任务。除首页外，每个页面都带着几个**只能从它进入的抽屉**，
抽屉的入口、数据、交互链全长在宿主页面上。拆开做有三个确定的坏处：

- **验不完整**：宿主页的验收标准里就有「点 X 打开 Y」这类条目，抽屉没做就没法验；
- **中间态断裂**：新版页面点开旧版抽屉，两套视觉语言正面撞在一起；
- **重复返工**：抽屉共享同一套通用规格（圆角/把手/遮罩/三键），分批做容易做出三个样。

所以**一簇一做、一簇一验、一簇上线**：

| 簇 | 成员 | 宿主与下钻关系 |
| --- | --- | --- |
| A ✅ | 01-首页 | 已完成上线 |
| B | **02-持仓页** + 03-持仓详情 + 04-持仓诊断 + 05-编辑抽屉 | 03 从逐股行点名称；04 从页头「诊断 N」或页名下小字行；05 从逐股行的基准股数/税率/每股股息三个可点域 |
| C | **06-股息日历** + 07-月明细 + 08-股息到账 | 07 从日历月点阵（首页月份点也进 07）；08 从 07 的可点行 |
| D | **09-收益明细** + 10-年度回顾 + 11-分享卡 + 12-年度回填 | 10 从年度行整行点击；12 从年度行「回填」；11 从 10 的页头「分享」 |
| E | **13-资金与交易** + 14-记一笔 + 15-交易 + 16-出入金 | 14 从首页「＋记一笔」；15/16 从 14 的选项，也从资金页对应流水行 |
| F | **17-公司基本面** + 18-选择公司 | 18 从基本面页公司名 |

**跨簇依赖（施工时注意）**：
- 05-编辑抽屉里的**负债弹窗**入口在首页净资产 label —— 首页已上线，做完 B 簇后线上首页的这个弹窗才统一。
- 07-月明细也能从**首页月份点**直接进；14-记一笔入口在**首页**。
  做 C / E 簇时顺带回归一下首页这两个入口。

**簇内推进节奏**：先把宿主页面的构图做出来出一次图给用户确认方向（不是完整验收，只看构图对不对），
再做簇内抽屉，最后整簇一起走验收流程、一起合并上线。
