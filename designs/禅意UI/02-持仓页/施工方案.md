# 持仓页 · 施工方案

以 定稿图.html 为唯一视觉标准（右侧功能对照单一并生效）。

## 结构（自上而下）
1. 页头 grid(56px/1fr/56px)：左裸返回箭头；中「持仓」11px 字距 5px 居中；右「诊断 N」11px（N>0 时数字 var(--up)）放入 grid 第三列——不许用 margin/justify hack 悬浮。页头下**无任何 rule**。
2. 页名下小字行 10px var(--hint) 居中：「行情 MM-DD HH:MM · N 项 · 点此打开诊断」，整行可点开诊断。
3. **居中 hero**：label「股票市值」→ 38px 主数 → 今日涨跌行（涨色）。数据取现 computeHoldings 汇总。
4. 仓位结构：2px 墨(核心)/金(打工)两段线 → 下方居中两项「核心仓 xx.x% / 打工仓 xx.x%」可点切换、金点选中 → 选中仓明细一行（市值/税后年化/股息率）→ 组合行（组合税后年化 金色 + 组合股息率）。**删除**旧三列小表(return-bar)与左右大块百分比+色块图例。
5. 逐股：节标「逐股」左 +「按市值 ↓」右（**单个文字按钮**轮换 市值→股息率→年股息，同字段再点切向；删除三 chip 盒）。
6. 逐股行两层：名称 14.5px/700+代码 10px hint（有在途/已公告股息名后 4px 金点，数据从 computeDividendCalendar().allDetails 组装）｜市值 14px/700 **墨色** 右；sub 行 10.5px muted：现价 · 税后年化 ¥ · 率 ｜ 权重右。四个可点域保留（名称→详情/年化→税率/率→每股股息/市值→数量或现金模式交易）。sub 行不得换行溢出。
7. 「展开全部 N 项」居中文字。

## 代码接线
render.js renderBucketsView（重写为居中结构）/renderReturnBar（并入组合行）/renderSortChips（改文字按钮）/renderDiagnosticsButton（进页头右槽）/getHoldingMarkup；main.js 绑定沿用。

## 禁止
黑 rule、chip 盒、行内金色市值、悬浮孤立字符、三列小表。
