# 持仓页 · 验收标准

## 核对方法（强制）
1. 浏览器打开本文件夹的 定稿图.html —— 这就是目标效果。
2. 启动应用（.claude/launch.json 的 dev 配置），打开对应界面。
3. **截图与定稿图并排逐元素对照**（不允许只用 DOM/computed 采样代替目检）。
4. 日/夜两态各对照一次（系统外观切换）。
5. 本页通过后向用户出示对比截图确认，才进入下一界面。

## 通用红线（任一违反即不通过）
- [ ] 无卡片边框、无图标圆底托、无 chevron 箭头、无分隔线/粗 rule（分区只靠留白）
- [ ] 无紫色/旧 Swiss 残留色；金色只出现在股息与选中记号
- [ ] 数字全部 tabular-nums；红涨绿跌语义正确
- [ ] 隐私掩码开启后本界面所有金额变掩码且不破版

## 本界面清单
- [ ] 页头三段 grid，「诊断 N」在右槽内不溢出版面，点击开诊断
- [ ] 38px 居中 hero「股票市值」存在且数据正确
- [ ] 结构区=细线+居中两项+明细行+组合行；点仓位可切换、金点跟随
- [ ] 排序为单文字按钮，点击轮换三字段、再点切升降
- [ ] 逐股行市值墨色、权重右下；sub 行一行放下不换行
- [ ] 造一条在途股息数据时行名后出现金点
- [ ] 展开/收起、下拉刷新、左缘返回不回归
