# 单字段编辑抽屉 · 验收标准

## 核对方法（强制）

**顺序不能反：先出图用眼睛看，再量数值。** 数值只验证得了你想到要验的维度。

1. 浏览器打开本文件夹的 定稿图.html —— 这就是目标效果（注意它是 390 画布快照，落地要按总则等比缩放，见 00-总则「定稿图怎么落地」）。
2. 起 dev server，出图：`npm run shot -- --shots=<页>,<页>:dark --statusbar`（用法见 00-总则/截图与验收.md）。
3. 用 Read 看 PNG，**与定稿图并排逐元素目检**，日/夜两态各一次。不允许用 DOM/computed 采样代替目检。
4. 再按下面清单逐项量数值。
5. 本页通过后向用户出示截图确认，才进入下一界面。

## 通用红线（任一违反即不通过）
- [ ] 无卡片边框、无图标圆底托、无 chevron 箭头、无分隔线/粗 rule（分区只靠留白）
- [ ] 无紫色/旧 Swiss 残留色；金色只出现在股息与选中记号
- [ ] 数字全部 tabular-nums；红涨绿跌语义正确，**金额后的百分比与金额同色**
- [ ] 隐私掩码开启后本界面所有金额变掩码且不破版
- [ ] **水平居中实测**：逐元素比中心偏移（≤2px），不能只信 CSS 写了 text-align:center —— 父级是 flex 时它对子项排列无效
- [ ] **尺寸随屏宽等比**：390 屏回到定稿值，430 屏放大约 1.103 倍（不是照搬绝对 px）
- [ ] **无旧层残留**：display / flex / height / min-height / font-size / border / ::before / ::after 均已显式覆盖
- [ ] **430×932 单屏不滚动**，底部动作键落在安全区之上；顶部留白不与状态栏重复
- [ ] 展示的比例与百分比接实时计算链，未写死定稿图上的示意值

## 本界面清单
- [ ] 四个弹窗均为此形制；输入聚焦时金线托底、无原生边框
- [ ] 保存后数据链路与旧版一致（含留空=未知语义）
- [ ] 抽屉规格全合
