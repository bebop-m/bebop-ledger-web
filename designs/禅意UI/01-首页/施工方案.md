## 落地实录（实际做法与返工点）

### 尺寸：全部等比缩放，不是照搬
首页 zen 层 56 处尺寸写成 `clamp(N*0.92px, N/390*100vw, N*1.35px)`。
390 屏精确回定稿值（hero 47px / 金线 214px / 内距 30px），430 屏等比放大（51.8 / 236 / 33）。
**返工点**：第一版照搬绝对 px，真机上整套设计缩小 9%，用户反馈「字号偏小」。
中途还错用过 `calc(100vw / 390)` 当缩放因子——长度除以无单位数仍是长度，`47px × 1.1px` 是非法的 px²，会让整层样式失效（字号全塌回 16px）。

### 画布：body 与 app-shell 交出边距
```css
body[data-active-page="home"] { padding: 0; }
body[data-active-page="home"] .app-shell { width: 100%; max-width: none; }
```
否则 body 的 22px + 页面 26px 叠加，430 屏内容区只剩 294px（应 378px）。

### 顶部留白：不与状态栏重复
状态栏样式是 `default`，页面从状态栏下方开始、`env(safe-area-inset-top)` 为 0。
定稿的 30px 设计留白降到 14px，真机总留白 = 状态栏 59pt + 15.4pt。
**返工点**：第一版按定稿写 30px，真机上成了 59+33=92pt，用户圈出「留白太多，整页偏下」。

### hero 必须显式 `display: block`
旧层把 `.home-hero` 设成了 `flex/column/align-items:flex-start`，此时 `text-align:center` 只作用于子项内部文本、不影响子项排列，**整个 hero 区贴左**。
**返工点**：这是最严重的一次漏检——我做了并排对照页、量出 17 项样式全等 + 10 个纵向锚点 0px 偏差，却没有一项检查水平对齐，用户一眼看出。

### 其余旧层残留（都要显式覆盖）
`.home-nav-item` 的 `flex:1`（五行平分空间、行高 49px→22px）、`.home-month` 的 `height:42px`、`.home-top` 的 `min-height:28px`、`.home-month.is-current` 的 `font-size:14px`、`.home-metrics` 的 border、`.home-month::before` 的旧刻度线。

### label 行盒要显式写 line-height
定稿图的 `.hero-label` / `.divi-label` 是 inline `<span>`，行盒被容器 16px 默认字号的 strut 撑到 **23px**（元素自身只有 14.5px）。落地改成 block 后只剩 14px，竖向节奏整体收紧 25px。显式写 `line-height: 23px` 才对得上。

### 红涨绿跌覆盖到百分比
`formatDailyPnl` 的输出从 `-¥996 (-0.38%)` 改成 `-¥996 · -0.38%`（定稿图的 `·` 分隔式），整段放进 `<strong>` 里随涨跌着色；汇率留在 strong 外保持 muted。
导航摘要的收益额与百分比同样着色（`.is-market-up/down`），其余摘要保持 muted。收益明细摘要去掉了「2026 · 」年份前缀。

### 数据一律接计算链
「本年股息 · N%」的 N = `getAnnualDividendOverview().receivedRatio`（已到账 ÷ 预计全年），不是定稿图的 58%。
主数用取整口径（`formatHudAmount`）以对齐定稿图的「¥42,180」无小数样式。
